**2. mapSpeed function:**
```typescript
export function mapSpeed(profileSpeed: number, minPx: number, maxPx: number): number {
  const base = minPx + ((profileSpeed - 1) / 9) * (maxPx - minPx);
  const variance = 0.9 + Math.random() * 0.2;
  return base * variance;
}
```

**3. wrapPosition function:**
```typescript
export function wrapPosition(x: number, y: number, w: number, h: number): { x: number; y: number } {
  return {
    x: x < 0 ? x + w : x > w ? x - w : x,
    y: y < 0 ? y + h : y > h ? y - h : y,
  };
}
```

Create `client/src/world/behaviors/` directory with 6 files. Each function is PURE — no PixiJS imports. Takes state + dt (seconds) + world bounds, returns new state object (spread input, override changed fields).

**walkingBehavior.ts** — `export function updateWalking(state: WalkingState, dt: number, world: WorldBounds): WalkingState`
- If `pauseTimer > 0`: decrement by dt*1000, return (entity is paused).
- If `walkTimer > 0`: move by vx*dt, vy*dt. Decrement walkTimer by dt*1000. Wrap position.
- If `walkTimer <= 0`: enter pause. Set pauseTimer to 500-1500ms (random). Pick new random heading. Set vx = cos(heading)*speed, vy = sin(heading)*speed. Set walkTimer to 2000-4000ms (random).
- Compute heading from vx/vy for rotation: `heading = Math.atan2(vy, vx)`.

**flyingBehavior.ts** — `export function updateFlying(state: FlyingState, dt: number, world: WorldBounds): FlyingState`
- Gradually turn: `heading += angularVelocity * dt`.
- Occasionally reverse arc direction: if random < 0.005 per frame (roughly every ~3 seconds at 60fps), flip `angularVelocity` sign.
- Update velocity: `vx = cos(heading) * speed`, `vy = sin(heading) * speed`.
- Move: x += vx*dt, y += vy*dt (using bobOriginY for the y base).
- Update bobPhase: `bobPhase += 2.5 * dt`. Compute y offset: `bobOriginY + sin(bobPhase) * 8`.
- Actually: bobOriginY tracks the "real" Y without bob. So: `bobOriginY += vy * dt`, then `y = bobOriginY + sin(bobPhase) * 8`.
- Wrap position on x and bobOriginY (not the bobbed y).

**rootedBehavior.ts** — `export function updateRooted(state: RootedState, dt: number, _world: WorldBounds): RootedState`
- Increment swayPhase: `swayPhase += 1.5 * dt` (radians/sec).
- Compute x: `x = originX + sin(swayPhase) * 3` (3px amplitude — very subtle).
- y unchanged.

**spreadingBehavior.ts** — `export function updateSpreading(state: SpreadingState, dt: number, _world: WorldBounds): SpreadingState`
- If `isACopy`: return state unchanged (copies don't spread).
- Decrement spawnTimer by dt*1000.
- If spawnTimer <= 0: set `pendingSpawn = true`, reset spawnTimer to spawnInterval.
- Otherwise: `pendingSpawn = false`.
- x, y unchanged (parent stays in place).

**driftingBehavior.ts** — `export function updateDrifting(state: DriftingState, dt: number, world: WorldBounds): DriftingState`
- Move x: `x += vx * dt`. Wrap x position.
- Update bobPhase: `bobPhase += 1.0 * dt`.
- Compute y: `y = bobOriginY + sin(bobPhase) * bobAmplitude`.
- bobOriginY is constant (no vertical drift, only bob).

**stationaryBehavior.ts** — `export function updateStationary(state: StationaryState, _dt: number, _world: WorldBounds): StationaryState`
- Return state unchanged. Literally `return state;`.

Define a shared `WorldBounds` type: `export interface WorldBounds { width: number; height: number; }` — put it in EntitySimulation.ts and import in each behavior file.

IMPORTANT: Do NOT use `Math.random()` inside behavior update functions for walk timer/pause duration — those values are set at state transitions. The walking function's "pick new direction" uses random, which is acceptable since it's a state transition, not per-frame noise. The flying arc reversal check (`random < 0.005`) is the one exception — it's a per-frame probability check, which is a standard game pattern.
