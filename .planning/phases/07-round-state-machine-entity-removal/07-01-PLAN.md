Modify WorldStage.ts to add the round state machine and entity removal infrastructure. All additions are to the existing `WorldStage` class:

**1. RoundPhase type (top of file, before class):**
```typescript
export type RoundPhase = 'idle' | 'analyzing' | 'simulating';
```
Note: No 'done' state. The original design had idle->analyzing->simulating->done->idle, but 'done' was set and immediately overwritten to 'idle' synchronously, making it unobservable. Instead, `_endRound()` transitions directly from simulating to idle.

**2. New private fields:**
- `_roundPhase: RoundPhase = 'idle'`
- `_dyingEntities = new Set<Container>()` — entities fading out, excluded from dispatch
- `_interactionMatrix: InteractionMatrix | null = null` — stored for Phase 8 consumption
- `_roundTimer: number | null = null` — setTimeout ID for 30s auto-end
- `_roundOverlay: RoundOverlay` — instantiated in constructor (`new RoundOverlay()`)
- `_onRoundPhaseChange: ((phase: RoundPhase) => void) | null = null` — callback for main.ts to react to phase changes

**3. Public getters:**
- `get entityCount(): number` — returns `this._entityStates.size`
- `get roundPhase(): RoundPhase` — returns `this._roundPhase`
- `get interactionMatrix(): InteractionMatrix | null` — returns `this._interactionMatrix`
- `set onRoundPhaseChange(cb: ((phase: RoundPhase) => void) | null)` — sets the callback

**4. `_gameTick` modifications (CRITICAL — order matters):**
- At the TOP of `_gameTick`, add: `if (this._roundPhase !== 'simulating') return;` — this freezes all entities when not in simulation phase (idle, analyzing). Per locked decision: entities freeze between rounds.
- Inside the for loop, BEFORE `dispatchBehavior`, add: `if (this._dyingEntities.has(container)) continue;` — dying entities are skipped entirely (REMV-02).

**5. `async startRound(): Promise<void>` method:**
- Guard: `if (this._roundPhase !== 'idle') return;` (prevents double-click)
- Guard: `if (this._entityStates.size === 0) return;` (no entities = no round)
- Set `this._roundPhase = 'analyzing'` and fire `_onRoundPhaseChange`
- Show spinner: `this._roundOverlay.showAnalyzingSpinner()`
- Collect profiles: `Array.from(this._entityProfiles.values())`
- Try/catch call to `fetchInteractions(profiles)`:
  - On success: store result in `this._interactionMatrix`
  - On error: log warning, create an all-ignore fallback using `{ entries: [] }` (empty entries = no interactions, consistent with server fallback behavior)
- Hide spinner: `this._roundOverlay.hideAnalyzingSpinner()`
- Set `this._roundPhase = 'simulating'` and fire `_onRoundPhaseChange`
- Start countdown: `this._roundOverlay.startCountdown(30)`
- Set timer: `this._roundTimer = window.setTimeout(() => this._endRound(), 30_000)`

**6. `_endRound(): void` method:**
- Clear timer: `if (this._roundTimer !== null) { clearTimeout(this._roundTimer); this._roundTimer = null; }`
- Stop countdown: `this._roundOverlay.stopCountdown()`
- Clear interaction matrix: `this._interactionMatrix = null`
- Transition directly to idle: `this._roundPhase = 'idle'` and fire `_onRoundPhaseChange`
  (No intermediate 'done' state — direct simulating->idle transition. Round summary overlay is deferred.)

**7. `removeEntity(container: Container): void` method (public — Phase 8 calls it for fight resolution):**
- Idempotent guard: `if (this._dyingEntities.has(container)) return;`
- Add to dying set: `this._dyingEntities.add(container)`
- Get label: `const label = this._entityLabels.get(container);`
- Fade-out animation using `app.ticker.add(fadeOut)` — mirror the existing fade-in pattern:
  - `elapsed` starts at 0, increments by `ticker.deltaMS`
  - `fadeDuration = 500` (0.5s per locked decision)
  - `alpha = Math.max(0, 1 - elapsed / fadeDuration)`
  - Set `container.alpha = alpha` and `if (label) label.alpha = alpha`
  - When alpha reaches 0: remove ticker callback, then cleanup:
    - Delete from all 5 maps: `_entityStates`, `_entityTextures`, `_entityProfiles`, `_entityLabels`, `_entitySpriteHeights`
    - Delete from `_dyingEntities`
    - `label?.destroy({ children: true })`
    - `container.destroy({ children: true })` — releases GPU texture references
    - **Order matters:** delete from maps BEFORE calling destroy() to prevent leaked texture references (Pitfall 1 from research)

Import `fetchInteractions` from `./fetchInteractions` and `RoundOverlay` from `./RoundOverlay` at the top of the file. Import `InteractionMatrix` from `@crayon-world/shared/src/types`.
