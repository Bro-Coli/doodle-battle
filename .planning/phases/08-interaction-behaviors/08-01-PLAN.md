  <behavior>
    Constants:
    - DETECTION_RANGE = 200 (pixels)
    - FIGHT_PROXIMITY_PX = 30 (pixels)
    - FIGHT_COOLDOWN_MS = 2000 (ms)
    - BEFRIEND_ARRIVE_RADIUS = 60 (pixels)

    seekPosition(sx, sy, tx, ty, speed, dt):
    - Returns {x, y, vx, vy} moving from (sx,sy) toward (tx,ty) at given speed
    - When distance < 1, returns original position with zero velocity
    - seekPosition(0, 0, 100, 0, 100, 1.0) -> x closer to 100, y stays 0

    fleePosition(sx, sy, tx, ty, speed, dt):
    - Returns {x, y, vx, vy} moving AWAY from (tx,ty)
    - fleePosition(50, 50, 100, 50, 100, 1.0) -> x < 50 (moves left, away from target at x=100)

    befriendPosition(sx, sy, tx, ty, speed, dt, arriveRadius=60):
    - Same as seek but speed scales down linearly within arriveRadius
    - At half arriveRadius, speed is ~half. At distance >> arriveRadius, full speed.
    - befriendPosition(0, 0, 200, 0, 100, 1.0) -> moves toward target at full speed
    - befriendPosition(0, 0, 30, 0, 100, 1.0) -> moves toward target at reduced speed (30/60 = half)

    resolveInteraction(selfContainer, entityStates, entityProfiles, dyingEntities, matrix, nameIdMap, detectionRange):
    - Returns {type, targetContainer, distance} for nearest non-ignore target within range
    - Returns null when: no targets in range, all relationships are 'ignore', entity not in matrix
    - Skips dying entities as potential targets
    - Picks nearest target when multiple qualify

    applyInteractionSteering(state, resolved, targetState, dt):
    - For 'chase': applies seekPosition at entity speed (default 80 if no speed field)
    - For 'flee': applies fleePosition at entity speed * 1.1
    - For 'fight': applies seekPosition at entity speed (approach the target)
    - For 'befriend': applies befriendPosition at entity speed * 0.5
    - Returns new EntityState spread with updated x, y (and vx, vy if applicable)
    - Preserves all archetype-specific fields untouched
  </behavior>
