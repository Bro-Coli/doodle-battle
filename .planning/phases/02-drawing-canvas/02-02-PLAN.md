1. Create `client/src/drawing/exportPng.ts`:
   - Import Application, Container, Graphics, Rectangle from pixi.js
   - Export `exportPng(app: Application, strokeContainer: Container, region: Graphics, padding = 16): string | null`:
     - If strokeContainer has no children, return null
     - Get bounding box of strokeContainer via `strokeContainer.getBounds()`
     - Create a Rectangle frame expanded by padding on all sides
     - Clamp frame to the region's world bounds so export never exceeds the drawing area
     - Cap resolution: pass `resolution: 1` to avoid high-DPI bloat (per research Pitfall 4)
     - Call `app.renderer.extract.canvas({ target: app.stage, frame, resolution: 1 })`
     - Convert result to data URL: `(canvas as HTMLCanvasElement).toDataURL('image/png')`
     - Return the data URL string
   - Note: If `renderer.extract.canvas()` returns a promise in the installed PixiJS version, make the function async and await it. Check the PixiJS v8 ExtractSystem API -- it may return `HTMLCanvasElement | Promise<HTMLCanvasElement>`. Handle both cases.

   - Import exportPng from './drawing/exportPng'
   - Wire submit button click:
     - Call `exportPng(app, drawingCanvas.strokeContainer, drawingCanvas.region)`
     - If result is null, do nothing (safety guard -- button should be disabled when empty anyway)
     - Log the data URL to console: `console.log('Exported PNG:', dataUrl.substring(0, 80) + '...')` (Phase 3 will POST this to /api/recognize)
     - After export, call `drawingCanvas.clear()` to auto-clear canvas per user decision
     - The onChange callback already handles disabling buttons after clear

3. Per user decision: submit button is already disabled when canvas is empty (wired in Plan 01 via onChange). No additional disable logic needed.
