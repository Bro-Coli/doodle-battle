Create a new module that bridges the Colyseus room's entity Schema state to WorldStage. This is the single integration point between networking and rendering for entities.

```typescript
// client/src/features/world/MultiplayerWorldBridge.ts

import type { Room } from '@colyseus/sdk';
import type { WorldStage } from './WorldStage';
import type { Archetype, EntityProfile } from '@crayon-world/shared/src/types';

/**
 * Lightweight shape matching EntitySchema fields received via onStateChange.
 * We don't import the actual Schema class (server-only with decorators).
 */
interface EntitySchemaLike {
  entityId: string;
  x: number;
  y: number;
  hp: number;
  name: string;
  archetype: string;
  teamId: string;
  vx: number;
  vy: number;
}
```

**Class: MultiplayerWorldBridge**

Constructor takes `worldStage: WorldStage` and stores it. Has a private `_room: Room | null` and `_knownEntityIds: Set<string>` for diffing.

**`connect(room: Room): void`**
- Store `_room = room`
- Set `worldStage.multiplayerMode = true`
- Register `room.onStateChange(callback)` where callback:
  1. Guard: `if (!room.state?.entities) return;` (Pitfall 3 from research)
  2. Get entities from state: `const entities = (room.state as { entities: Map<string, EntitySchemaLike> }).entities`
  3. Build a `currentIds = new Set<string>()` from the entities map
  4. For each entity in the map:
     - If NOT in `_knownEntityIds`: this is a NEW entity — call `worldStage.spawnFromSchema(entityId, profile, schema.x, schema.y)` where profile is constructed from Schema fields: `{ name: schema.name, archetype: schema.archetype as Archetype, traits: [], role: '', speed: 0 }` (traits/role/speed not needed for rendering)
     - Add to `_knownEntityIds`
  5. Call `worldStage.applyPositions(positionMap)` with all entity positions/velocities/hp
  6. For each id in `_knownEntityIds` that is NOT in `currentIds`: entity was removed server-side — call `worldStage.removeEntityById(entityId)` and delete from `_knownEntityIds`

**`disconnect(): void`**
- If `_room`, call `_room.onStateChange.remove(callback)` (store callback reference for removal)
- Set `worldStage.multiplayerMode = false`
- Clear `_knownEntityIds`
- Set `_room = null`

**`sendSpawnEntity(entityId: string, profile: EntityProfile, x: number, y: number, teamId: string): void`**
- Sends `room.send('spawn_entity', { entityId, name: profile.name, archetype: profile.archetype, speed: profile.speed, x, y, teamId })` to server
- This is called by the game flow when recognition completes — the server creates the EntitySchema

**`sendInteractionMatrix(matrix: InteractionMatrix): void`**
- Sends `room.send('interaction_matrix', matrix)` to server

Export the class as a named export. Do NOT export a singleton — the caller creates and manages the instance.

This module has NO PixiJS imports (it only talks to WorldStage via its public API). It has NO Colyseus Schema imports (uses duck-typed EntitySchemaLike). This keeps it lightweight and testable.
