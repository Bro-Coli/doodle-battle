**Then implement in GameRoom.ts:**

1. Add `EntitySchema` class with `@type` decorators for: `entityId` (string), `x` (number), `y` (number), `hp` (number, default 1), `name` (string), `archetype` (string), `teamId` (string), `vx` (number), `vy` (number). The `vx`/`vy` fields enable client-side sprite orientation without inferring from position deltas.

2. Add `entities` MapSchema to `GameState`:
   ```typescript
   @type({ map: EntitySchema }) entities = new MapSchema<EntitySchema>();
   ```

3. Add private fields to GameRoom (NOT in Schema — server-only):
   - `_entityStates: Map<string, EntityState>` — full behavioral state per entity
   - `_entityProfiles: Map<string, EntityProfile>` — profiles for interaction resolution
   - `_interactionMatrix: InteractionMatrix | null` — from client via room message
   - `_fightCooldowns: Map<string, number>` — per attacker:target pair
   - `_nameIdMap: Map<string, string>` — entity name to integer ID
   - `_dyingEntities: Set<string>` — entities in removal (skip in tick)

4. In `onCreate`, after existing code:
   - Initialize all private fields (Maps, null matrix, empty Set)
   - Register `this.setSimulationInterval((deltaTime) => this._tick(deltaTime), 50)` for 20Hz tick
   - Register message handlers:
     - `'spawn_entity'`: calls `_handleSpawnEntity(client, msg)` — msg contains `{ entityId, name, archetype, teamId, speed, x, y }`
     - `'interaction_matrix'`: calls `_handleInteractionMatrix(client, msg)` — msg contains `InteractionMatrix`
     - `'remove_all_entities'`: clears all entity state (for round reset)

5. Implement `_tick(deltaMS: number)`:
   - Convert `deltaMS / 1000` to `dt` seconds
   - Import and use `WORLD_BOUNDS` from shared
   - Calculate `worldDiag`, `detectionRange`, `fightProximity` from WORLD_BOUNDS
   - Decrement `_fightCooldowns` by `deltaMS`; delete expired
   - Iterate `_entityStates` with `for...of`:
     - Skip if in `_dyingEntities`
     - Determine if movable (not rooted/stationary) and resolve interactions if matrix exists
     - Call `dispatchBehavior(state, dt, WORLD_BOUNDS)` or `applyInteractionSteering` based on resolved interaction
     - Fight contact check: if chase/fight and distance < fightProximity, call `_handleFightContact`
     - Bounce off borders using entity size (use 40px default half-size since actual sprite size is client-only)
     - Handle spreading pendingSpawn: create new EntitySchema with `crypto.randomUUID()`, init new EntityState as copy
     - Update EntitySchema: `schema.x = newState.x; schema.y = newState.y; schema.vx = newState.vx ?? 0; schema.vy = newState.vy ?? 0;`
     - Collect removals in `toRemove: string[]`
   - AFTER loop: apply removals — delete from `_entityStates`, `_entityProfiles`, `state.entities`, `_dyingEntities`

6. Implement `_handleFightContact(attackerId: string, targetId: string)`:
   - Cooldown key: `${attackerId}:${targetId}` — skip if in `_fightCooldowns`
   - Get EntitySchema for target, decrement HP
   - If HP <= 0: add to `_dyingEntities`, push to toRemove (or delete immediately since this is called during tick — use the toRemove collector from `_tick`)
   - Actually: the cleanest approach is to have `_handleFightContact` just set HP to 0 and add to `_dyingEntities`. The `_tick` loop checks `_dyingEntities` and skips those entities. A separate pass after the main loop cleans up dying entities with HP <= 0. This avoids mutation during iteration.
   - Set cooldown: `_fightCooldowns.set(cooldownKey, FIGHT_COOLDOWN_MS)`

7. Implement `_handleSpawnEntity(_client: { sessionId: string }, msg: unknown)`:
   - Validate msg has required fields (entityId, name, archetype, speed, x, y, teamId)
   - Create `EntitySchema`, set all fields
   - Create `EntityState` via `initEntityState(archetype, speed, x, y)`
   - Add to `state.entities.set(entityId, schema)` and `_entityStates.set(entityId, state)` and `_entityProfiles`

8. Implement `_handleInteractionMatrix(_client: { sessionId: string }, msg: unknown)`:
   - Store `msg` as `this._interactionMatrix`
   - Build `_nameIdMap` from the matrix entries

9. Implement `_buildNameIdMap(profiles: EntityProfile[])`: same logic as current WorldStage version — unique names mapped to string integer IDs.

Use `crypto.randomUUID()` (Node.js built-in) for entity IDs on spawn. Import `EntityState`, `SpreadingState`, `initEntityState`, `dispatchBehavior`, `WORLD_BOUNDS` from shared simulation. Import `resolveInteraction`, `applyInteractionSteering`, `DETECTION_RANGE_FRACTION`, `FIGHT_PROXIMITY_FRACTION`, `FIGHT_COOLDOWN_MS` from shared interactionBehaviors. Import `EntityProfile`, `InteractionMatrix` from shared types.

The `resolveInteraction` function is generic over `<T>` — on server, `T = string` (entity IDs). The existing client code uses `T = Container`. This is already supported by the generic signature.
