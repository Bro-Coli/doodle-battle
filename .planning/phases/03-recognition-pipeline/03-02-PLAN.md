  <action>
    **1. Create `client/src/recognition/recognizeApi.ts`:**
    - Export `recognizeDrawing(dataUrl: string): Promise&lt;EntityProfile&gt;`.
    - POST to `/api/recognize` with JSON body `{ imageDataUrl: dataUrl }`, Content-Type application/json.
    - On success (200), parse JSON as EntityProfile and return.
    - On non-200 or network error: retry once after ~1000ms delay (silent retry per locked decision). On second failure, throw an Error with message from response or "Recognition failed".
    - Import `EntityProfile` from `@crayon-world/shared`.

    **2. Create `client/src/recognition/RecognitionOverlay.ts`:**
    Export a class `RecognitionOverlay` that manages HTML overlay elements appended to `document.body`. All use `position: fixed` following the toolbar pattern from style.css.

    Methods:
    - `showSpinner()`: Show a centered spinner overlay (semi-transparent backdrop over canvas). Start a 10-second soft timeout timer — after 10s, add "Taking longer than expected..." text below the spinner.
    - `hideSpinner()`: Remove spinner overlay and clear timeout timer.
    - `showCard(profile: EntityProfile, onDismiss: () => void)`: Hide spinner first. Show a centered card with: entity name (large), archetype badge, traits as comma-separated list, role text. Start 5-second auto-dismiss timer. Click anywhere on the card also dismisses. On dismiss (auto or click): remove card, call `onDismiss` callback. Ensure onDismiss is called only once (guard against double-fire from timer + click).
    - `showError(message: string, onDismiss: () => void)`: Hide spinner first. Show a toast/banner above the drawing region with error message and a "Retry" button. Clicking the dismiss X or the Retry button removes the toast and calls `onDismiss`. The Retry button is just a dismiss — the actual retry happened silently in recognizeApi.
    - `showMockBadge()`: Show a subtle "Mock Mode" badge in the bottom-left corner. Call once at init if mock mode is detected.

    Each method creates elements on demand and removes them on hide/dismiss. No persistent DOM — create when showing, remove when hiding.

    **3. Update `client/src/style.css`:**
    Add CSS classes for all overlay components:
    - `.spinner-overlay`: fixed, inset 0, flex center, semi-transparent white backdrop, z-index 20
    - `.spinner`: 40px circle, border animation, CSS `@keyframes spin`
    - `.spinner-timeout-msg`: text below spinner for "Taking longer than expected..."
    - `.result-card`: fixed, centered, white card with shadow, rounded corners, z-index 30, padding. Name large (1.5em), archetype as small uppercase badge, traits italic, role normal text. Cursor pointer (clickable to dismiss).
    - `.error-toast`: fixed, top center (below toolbar ~80px from top), red-tinted background, white text, z-index 30, with close X button and Retry button
    - `.mock-badge`: fixed, bottom-left, small, semi-transparent, z-index 10, subtle gray text

    **4. Add mock mode status endpoint to `server/src/routes/recognize.ts`:**
    Append a GET handler (Plan 01 work is complete by wave 2, so this is a safe append):
    `router.get('/status', (_req, res) => { res.json({ mockMode: isMockMode() }); });`
    This lets the client detect mock mode at page load without waiting for a recognition call.

    **5. Rewrite submit handler in `client/src/main.ts`:**
    - Import `recognizeDrawing` from `./recognition/recognizeApi`.
    - Import `RecognitionOverlay` from `./recognition/RecognitionOverlay`.
    - Create `const overlay = new RecognitionOverlay()` after toolbar setup.
    - At init, fetch `GET /api/recognize/status` — if `mockMode` is true, call `overlay.showMockBadge()`. Swallow errors (badge is non-critical).
    - Create helper functions `disableAllToolbar()` and `enableAllToolbar()` that set `.disabled` on submitBtn, clearBtn, undoBtn, and all thickness buttons. `enableAllToolbar()` must call `syncButtonState()` afterward to restore correct empty/non-empty state for Submit/Clear/Undo.
    - Replace the submit click handler with an async version:
      ```
      submitBtn.addEventListener('click', async () => {
        const dataUrl = exportPng(app, drawingCanvas.strokeContainerRef, drawingCanvas.region);
        if (dataUrl === null) return;
        // DO NOT clear canvas here — clear on card dismiss (locked decision)
        overlay.showSpinner();
        disableAllToolbar();
        try {
          const profile = await recognizeDrawing(dataUrl);
          overlay.showCard(profile, () => {
            drawingCanvas.clear();
            enableAllToolbar();
          });
        } catch {
          overlay.showError('Recognition failed. Try again.', () => {
            enableAllToolbar();
          });
        }
      });
      ```
    - Remove the old `console.log` and `drawingCanvas.clear()` from the submit handler.
