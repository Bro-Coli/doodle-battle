1. Create `client/vite.config.ts` with:
   - Vite defineConfig
   - server.proxy: '/api' -> 'http://localhost:3001' with changeOrigin: true
   - No framework plugins needed (vanilla TS)

2. Create `client/index.html`:
   - Standard HTML5 boilerplate, lang="en"
   - Title: "Crayon World"
   - Link to src/style.css (or let Vite handle it via main.ts import)
   - Script tag: `<script type="module" src="/src/main.ts"></script>`
   - Body contains only a div#app (PixiJS canvas appends to document.body)
   - No login forms, no auth, no install prompts

3. Create `client/src/style.css`:
   - Reset: margin 0, padding 0, overflow hidden on html and body
   - body: width 100vw, height 100vh, background #FFFFFF
   - Toolbar styles: fixed position overlay at top-center of viewport
   - Toolbar buttons: styled but clearly disabled (opacity, cursor: not-allowed)
   - Use a clean, minimal game UI style — no heavy frameworks

4. Create `client/src/main.ts`:
   - Import './style.css'
   - Import { Application } from 'pixi.js'
   - Create async function init():
     a. Create new Application()
     b. await app.init({ resizeTo: window, autoDensity: true, background: '#FFFFFF' })
        IMPORTANT: Do NOT pass options to constructor. Must use await app.init() — PixiJS v8 pattern.
     c. document.body.appendChild(app.canvas)
   - Create toolbar with three HTML buttons: Submit, Clear, Undo
     - All three disabled (disabled attribute set)
     - Positioned as a fixed overlay bar at top of canvas
     - Submit button will be used in Phase 3 to trigger recognition
     - Clear and Undo will be wired in Phase 2
   - Call init() (not top-level await — wrap in async function per research pitfall warning)
   - Do NOT use top-level await (causes Vite production build failures)

5. Verify: Run `pnpm dev:client` and confirm canvas loads at localhost:5173
